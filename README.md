## 專案簡介

這個專案是一個由 AI Agent 自主設計與開發的實驗性平台，旨在處理、分析並呈現社群媒體上的推文資料。透過機器學習和自然語言處理技術（或其模擬），我們的 AI Agent 能夠理解推文內容並提取關鍵資訊，目前正處於功能迭代與增強階段，致力於提供更智能、更高效的資料整理服務。

## 目前任務：推文數由大到小排序功能

目前，AI Agent 正專注於開發一項關鍵功能：根據推文的「熱門程度」或「推文數」進行由大到小的排序。此功能旨在協助使用者快速識別最受關注的推文。

### 功能細節與排序規則：

為了實現精確的排序，AI Agent 將遵循以下特殊規則來評估推文數：

1.  **「爆」的定義：** 將推文數中出現的特殊標記 `爆` 視為 `100`。
2.  **負評「X」的定義：** 將以 `X` 開頭的負評（例如 `X1`, `X2`, `X3` 等）統一視為 `0`。
3.  **其他數字：** 除上述兩種情況外，其他數字則按其實際數值進行排序。

### 輸出格式：

排序結果將以清晰的**表格形式**輸出，確保推文數最高的項目排在最前面，方便使用者一目瞭然地查看熱門推文。

**範例輸入資料 (假設 AI Agent 處理的原始資料)：**

| 內容         | 推文數 |
| :----------- | :----- |
| AI 專案真有趣 | 15     |
| 開發新功能     | X1     |
| 學習新技術     | 爆     |
| 今日天氣真好   | 5      |
| 週末活動規劃   | X3     |
| 程式碼優化     | 20     |

**預期輸出結果 (經 AI Agent 排序後)：**

| 內容         | 轉換後推文數 |
| :----------- | :----------- |
| 學習新技術     | 100          |
| 程式碼優化     | 20           |
| AI 專案真有趣 | 15           |
| 今日天氣真好   | 5            |
| 開發新功能     | 0            |
| 週末活動規劃   | 0            |

## AI Agent 如何運作

本專案的核心由一個或多個 AI Agent 驅動。這些 Agent 負責：

*   **資料收集與預處理：** 從指定來源（模擬或實際）獲取推文資料，並進行清洗。
*   **功能實現與迭代：** 根據專案需求，自主分析並生成程式碼，實現新的功能，如目前的排序功能。
*   **測試與優化：** 對開發的功能進行自動化測試，並根據結果進行優化和修正。
*   **學習與適應：** 從每次任務的執行中學習，不斷提升自身的開發能力與效率。
