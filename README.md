# 🤖 由 AI Agent 開發的網頁抓取器：錯誤修復與強化

此專案是一個由人工智慧代理（AI Agent）獨立設計與開發的實驗性專案，旨在探索與實踐自動化網頁資料抓取的可能性。透過不斷的迭代與學習，AI Agent 嘗試克服網頁抓取中遇到的各種挑戰，並持續優化其抓取策略與穩定性。

## 🎯 當前任務：修復 ConnectionResetError 並強化穩定性

在資料抓取過程中，AI Agent 頻繁遭遇 `ConnectionResetError`，此錯誤嚴重影響了抓取任務的穩定性與效率。因此，AI Agent 將集中資源修復此問題，並強化抓取機制的韌性。

為了解決 `ConnectionResetError` 並提升整體抓取成功率，AI Agent 將實施以下策略：

### 🛠️ 核心修復與強化策略

1.  **完整的 User-Agent 模擬**
    *   **問題：** 許多網站會透過檢查 `User-Agent` 標頭來識別並阻擋自動化請求。不完整的或不常見的 `User-Agent` 容易觸發網站的反爬蟲機制。
    *   **解決方案：** AI Agent 將確保為每個 HTTP 請求設定完整的 `User-Agent` 標頭，精確模擬真實的 Chrome 瀏覽器行為。這將使得請求看起來更像來自於普通用戶，從而降低被識別和阻擋的風險。
    *   **範例 User-Agent (模擬 Chrome 瀏覽器):**
        ```
        User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36
        ```

2.  **正確處理 18+ 內容驗證與 Cookies 管理**
    *   **問題：** 許多網站含有需要年齡驗證的內容。如果未能正確處理驗證過程中的 `cookies`，抓取器將無法訪問目標內容，或被迫重複進行驗證。
    *   **解決方案：** AI Agent 將實現一套智能的 `cookies` 管理機制。針對需要年齡驗證的網站，AI Agent 會在首次驗證成功後，妥善地保存並在後續的所有請求中正確攜帶這些 `cookies`。這將維持會話狀態，確保抓取器能夠無縫地訪問受限內容，避免重複驗證導致的效率低下和潛在的錯誤。

3.  **抓取失敗時的隨機延遲與重試機制**
    *   **問題：** `ConnectionResetError` 或其他網路暫時性錯誤可能隨機發生。立即重試可能再次失敗，且過快的請求頻率可能加劇伺服器壓力或觸發阻擋。
    *   **解決方案：** 當抓取請求失敗（特別是因 `ConnectionResetError` 或其他網路問題）時，AI Agent 將啟用自動重試機制。每次重試前，系統會引入一個隨機的延遲時間（`time.sleep`），以模擬人類瀏覽行為，減少伺服器壓力，並給予伺服器足夠的時間恢復。隨機延遲有助於避免被檢測為固定模式的自動化程式，從而提高重試的成功率。
    *   **實作考量：** 延遲時間將設定在一個合理的範圍內（例如，2 到 10 秒之間），並可根據實際情況進行動態調整。

### 📊 進度與結果

AI Agent 將持續監控其抓取任務，記錄 `ConnectionResetError` 的發生頻率，並分析上述策略實施後的成效。所有改進和調整都將基於實際運行的數據進行。

## 🚀 如何運行 (觀察模式)

此專案主要由 AI Agent 自主運行和迭代。然而，如果您想在本地觀察 AI Agent 的行為或進行手動測試，可以按照以下步驟：

1.  **克隆專案：**
    ```bash
    git clone [您的專案_URL]
    cd [您的專案_目錄]
    ```
2.  **安裝依賴：**
    ```bash
    pip install -r requirements.txt
    ```
3.  **運行 AI Agent 主程式：**
    ```bash
    python main.py
    ```
    （請注意，`main.py` 僅為示例，實際入口點可能有所不同。）

**配置說明：**
您可能需要在環境變數中設定目標網站的 URL，或在 `config.py` / `settings.json` 等配置檔中提供相關參數，例如：
*   `TARGET_URL`: 目標網站的基礎 URL
*   `MAX_RETRIES`: 最大重試次數
*   `MIN_DELAY`, `MAX_DELAY`: 隨機延遲的最小與最大秒數
